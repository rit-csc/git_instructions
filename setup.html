Setting up a repository

Configuring your name and email

Before using Git on any system for the first time, it is a good idea to set your name and email address, since Git will include this information in each commit message you author. Follow these steps:

  1. Set your name by using this command: 
    $ git config --global user.name "<Your Name>"
  2. Set your email as follows: 
    $ git config --global user.email <your_email_address>

If you plan to develop on the CS systems

If you intend to write your code on the department's systems (e.g. while connected via SSH), all you need to do is initialize a standard Git repository. To do this, follow these steps.

  1. Make sure you've set your name and email, as detailed in the above section. To check, look for this information in the output of the command: 
    $ git config --global -l
  2. Move into the folder where you keep your work for the course, for example by running the command: 
    $ cd ~/Courses/CS/CS1/Labs
  3. Create a repository in a new folder of your choice. For instance, to create one called Lab1, run: 
    $ git init Lab1
  4. This should result in a message being printed in the terminal, eg: 
    Initialized empty Git repository in ~/Courses/CS/CS1/Labs/Lab1/.git/
    (Note: the exact path may differ based on your exact setup!!)
  5. On the offchance that this message was not printed, there is another way to check whether your repository was successfully created. To do this, navigate into the new directory and list its contents with the following series of commands: 
    $ cd Lab1
$ ls -a

    Now look for the hidden .git folder. This folder identifies the directory as a Git repository. If you see it, you know the init was successful!

If you plan to develop on your own machine

If you intend to write your code on your own system, you have to create an upstream on the CS systems and clone it to a standard Git repository on your own machine. This is done as follows:

  1. Working on a CS department system, move into the folder where you keep your work for the course, for example by running the command: 
    $ cd ~/Courses/CS/CS1/Labs
  2. Create a bare repository in a new folder of your choice (or use a repository that you have already created). To create a bare repo called Lab1.git, run: 
    $ git init --bare Lab1.git
  3. This should result in a message being printed in the terminal, eg: 
    Initialized empty Git repository in ~/Courses/CS/CS1/Labs/Lab1.git/

    (Note: the exact path may differ based on your exact setup!!)
  4. On your own system, download Git from the project website and install the package.
  5. If you're on a Windows system, locate and launch the newly-installed "Git Bash" application; otherwise, simply open a terminal (e.g. on Mac OS X, click the magnifying glass in the top right corner of the screen and type "Terminal").
  6. In the terminal on your own system, move into the directory where you keep your coursework, for example: 
    $ cd ~/Courses/CS/CS1/Labs
  7. Clone the repository you created earlier on the CS systems. To do this for an upstream you called Lab1.git, run the following command and then enter your CS password: 
    $ git clone {your-username}@glados.cs.rit.edu:Courses/CS/CS1/Labs/Lab1
  8. This should result in a message being printed in the terminal, eg: 
    Cloning into 'Lab1'...
  9. On the offchance that this message was not printed, there is another way to check whether your repository was successfully created. To do this, navigate into the new directory and list its contents with the following series of commands: 
    $ cd Lab1
    $ ls -a
  10. Now look for the hidden .git folder. This folder identifies the directory as a Git repository. If you see it, that means the clone was successful!

Regarding code hosting services and public repositories

There are many online services that offer Git repository hosting, the most popular being GitHub and BitBucket. For the sake of simplicity, the CS department recommends that you store your code on department systems and access it remotely as-needed. If, however, you decide to use a third-party service, carefully ensure that your hosted repositories are private! If you publish course code to a public repository, you can be penalized for cheating pursuant to RIT's Academic Integrity Policy and the department Policy on Academic Honesty.
